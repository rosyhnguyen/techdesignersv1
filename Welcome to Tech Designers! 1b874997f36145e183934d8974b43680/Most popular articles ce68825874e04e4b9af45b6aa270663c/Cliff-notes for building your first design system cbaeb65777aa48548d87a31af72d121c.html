When I first created my design system, I didn’t know where to start and how to start. It took me almost 4 weeks to finish my first design system and countless time to update the new design components to the development team — I guess they were tired of changes from my side.

I’m collecting all the resources I have used and put them all into these cliff-notes. This can save you a lot of time.

What is design system?

A design system is a collection of reusable components, guided by clear standards, that can be assembled together to build any number of applications.

Before you start creating your design system, read this critical guide: Build a system that provides a unified set of UX, design rules and patterns.

Color

Color is a tricky part, especially when you have to work with hot color like red, orange or strong tones like purple. Unfortunately, you can’t choose your company color. If the founder likes red for the logo, you’ll have no other way but to go ahead with red design system. However, a balancing secondary colour is a good blend. For example, red goes well with dark indigo. To check which colour goes well with what, check out Materio.

ColorBox is a cool tool to define all the shades you need for each color. This site also help you check if the colour passes accessibility standard. Every color between 0–50 is accessible (4.5:1) on black, and every color 60–100 is accessible (4.5:1) on white.

If you are considering using gradient, uiGradients has awesome gradient samples. The best gradients are created by nature. Follow what nature has created and you’ll get great gradients.
